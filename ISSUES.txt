BUGS — things that are factually wrong
==============================================================

[BUG-1] EYE > Overlay label wrong during breaks
  Files: src/renderer/public/overlay.html:88-93
  Problem: checks pomodoroState === 'short-break' || 'long-break' — these values never exist.
           pomodoroState is only 'work'|'break'|'idle'|'paused'.
           Break type lives in pomodoroMode, not pomodoroState.
  Fix: change condition to pomodoroState === 'break'; optionally also send pomodoroMode
       in the overlay:update payload to distinguish short vs long.

[BUG-2] ✅ FIXED — EYE > Blink rate underreports for the first 60s of every session
  Files: src/renderer/src/hooks/useEyeTracker.js:142
  Problem: setBlinkRate(blinkTimesRef.current.length)
           blinkTimesRef is filtered to last 60s window. If session has run 10s and 2 blinks
           occurred, reports 2 BPM. Real rate is 12 BPM.
           Corrupts focus score for all short sessions and the first minute of every session.
  Fix: normalize — Math.round(count * 60000 / Math.min(elapsed, 60000))
       where elapsed = Date.now() - trackingStartTime.
       Needs a trackingStartTimeRef initialized in startTracking().

[BUG-3] ✅ FIXED — POMODORO > Away time in saved session data is cumulative, not per-session
  awayStartRef added to usePomodoro; snapshotted when start() is called from idle.
  Session data uses awaySecondsRef.current - awayStartRef.current in both timer and skip().

[BUG-4] ✅ FIXED — DATA > Sessions not loaded on app restart — stats always zero
  Added appendSessionJson/loadSessionsFromJson to persistence.js (atenttion-sessions.json).
  data:saveSession now writes both markdown (human) and JSON (machine) in parallel.
  Added data:loadSessions IPC + applySessionHistory store action + startup load in App.jsx.
  todayFocusSeconds is recomputed from the loaded sessions array filtered to today's date.

[BUG-5] ✅ FIXED — DATA > App usage data never persisted
  Added saveAppUsage/loadAppUsage to persistence.js (atenttion-app-usage.json, date-stamped).
  Saved on every session end (via saveSession in usePomodoro) + every 60s in App.jsx.
  Loaded on startup if stored date matches today (stale data from yesterday is ignored).
  setAppUsage store action restores appUsageFocus + appUsageBreak on load.

DESIGN PROBLEMS — work but are bad
==============================================================

[DESIGN-1] PERF > usePomodoro subscribes to entire store
  Files: src/renderer/src/hooks/usePomodoro.js:25,42-47
  Problem: const store = useStore() — re-renders on every state change in the whole app.
           Every blink update, every timer tick triggers 6 ref-sync effects.
           Approximately 1-2 re-renders/sec while running, multiplied by all effects.
  Fix: remove const store = useStore(). Read values via useStore.getState() inside
       callbacks (already done for most things). Delete all 6 ref-sync useEffects.
       Replace stateRef.current etc. with direct useStore.getState().pomodoroState calls.

[DESIGN-2] ✅ FIXED — UX > sessionsCompleted and sessions.length are the same number — one is redundant
  StatsPage and FocusPage now derive today's session count from sessions[] (persisted JSON)
  instead of sessionsCompleted (runtime-only counter). Label updated to "Sessions Today".
  sessionsCompleted stays in the store for Pomodoro mode-rotation logic only.

[DESIGN-3] ✅ FIXED — DATA > App usage doesn't reset at midnight
  storedDateRef added to useAppTracker; compared to today on each 2s poll.
  On date change: calls resetDailyMetrics() (new store action) + saves zeroed usage JSON.

[DESIGN-4] ✅ FIXED — BUG-RISK > SystemPage and App.jsx both independently load getCurrentWallpaper()
  originalWallpaper added to Zustand store. App.jsx loads it once on startup + syncs ref.
  Wallpaper effect checks store before falling back to IPC. SystemPage reads from store,
  removed its local state + its own getCurrentWallpaper() IPC call entirely.

[DESIGN-5] UX > Wallpaper feature is always on, no way to disable
  Files: src/renderer/src/App.jsx:57-82
         src/renderer/src/components/Settings.jsx
  Problem: Desktop wallpaper changes to #111111 on every Pomodoro start. No toggle.
           Users who don't want this can't turn it off.
  Fix: add enableFocusWallpaper: true preference. Show toggle in Settings.
       Gate the entire wallpaper effect block behind this flag.
